# ply-flow

O loop vive no AGENTS.md. Esta skill cobre o que não cabe lá — leia só a
reference que a situação pede:

- Task grande demais → `references/decompose.md`
- O que conta como "done" → `references/dod.md`
- Antes de escrever código novo → `references/ladder.md`

Regra de ouro: o brief é o contrato. AC não testável é bug do AC — conserte a
task (e registre lesson), não invente comportamento.
